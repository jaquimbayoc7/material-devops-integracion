🛠️ DevOps Task Manager API

Proyecto integrador para diseñar e implementar un pipeline completo de CI/CD usando FastAPI, Docker, Jenkins y Kubernetes. Cada commit al repositorio desencadena automáticamente pruebas, construcción de imagen, publicación en registry y despliegue en un clúster Kubernetes local.

🐍 FastAPI + SQLite 🐳 Docker multistage 🤖 Jenkins LTS ⎈️ Kubernetes 📦 Docker Hub 🧪 pytest · 11 tests ⏱️ 3 semanas

1. Stack Tecnológico

El proyecto combina cuatro pilares del ecosistema DevOps moderno. Cada herramienta cumple un rol específico en la cadena de entrega de software.

🐍

FastAPI (Python 3.10+)

Framework moderno para APIs REST de alto rendimiento con documentación interactiva automática en /docs. Aquí construimos el CRUD de tareas que recorre todo el pipeline.

Persistencia: SQLite con SQLAlchemy ORM. Incluye seed de 10 tareas iniciales (seed.py).

🐳

Docker

Conteneriza la aplicación en una imagen inmutable multistage: etapa builder que instala dependencias y etapa runtime slim con usuario sin privilegios.

docker-compose levanta el entorno de desarrollo con hot-reload.

🤖

Jenkins

Servidor de automatización open-source que ejecuta el pipeline declarativo. Detecta cambios en GitHub via SCM polling y corre tests, build, push y deploy automáticamente.

Corre en contenedor Docker vía docker-compose en jenkins/.

⎈️

Kubernetes

Orquestador de contenedores que gestiona el ciclo de vida de los pods y escala automáticamente con el HPA (Horizontal Pod Autoscaler) al superar el 60% de CPU.

Docker Desktop con Kubernetes habilitado — sin necesidad de nube.

📦

Docker Hub

Registro público de imágenes. Jenkins sube la imagen con cada pipeline exitoso y Kubernetes la descarga para actualizar los pods sin downtime.

Cuenta gratuita en hub.docker.com.

🐙

GitHub

Repositorio de código fuente y punto de partida del pipeline. Jenkins monitorea con SCM polling: cada commit dispara automáticamente una nueva ejecución.

Contiene también el Jenkinsfile, Dockerfile y manifiestos Kubernetes.

2. Arquitectura del Sistema

El siguiente diagrama muestra el flujo completo: desde el git push hasta que el nuevo código corre en Kubernetes con actualización sin downtime.

📖 Cómo leer el diagrama:
  1. ① El desarrollador hace git push al repositorio en GitHub.
  2. ② Jenkins detecta el cambio (SCM polling) y ejecuta el Jenkinsfile con 5 stages.
  3. ③ El stage Build construye la imagen Docker y el stage Push la sube a Docker Hub.
  4. ④ El stage Deploy ejecuta kubectl apply -f k8s/ y Kubernetes actualiza los pods sin downtime.

3. El Pipeline CI/CD

El Jenkinsfile define los 5 stages del pipeline como código versionado en el repositorio (Pipeline as Code). Jenkins lo ejecuta automáticamente con cada commit detectado.

💡 Pipeline as Code:
El Jenkinsfile vive dentro del mismo repositorio que el código fuente. Permite versionar el pipeline, revisarlo en pull requests y replicarlo en cualquier Jenkins sin configuración manual. Es una práctica fundamental de DevOps.

4. Plan de 3 Semanas

🐍 Semana 1 App + Docker ✓ Completada

Construir la aplicación FastAPI y contenerizarla con Docker. La API corre localmente con 11 tests en verde y seed de base de datos incluido.

  1. ✅ Scaffolding del repositorio (.gitignore, requirements.txt, README.md)
  2. ✅ FastAPI CRUD completo de tareas (GET / POST / PUT / DELETE)
  3. ✅ Endpoint /health para liveness y readiness probes de Kubernetes
  4. ✅ Tests con pytest — 11 tests en verde (crear, listar, actualizar, eliminar)
  5. ✅ Dockerfile multistage (etapa builder + etapa runtime slim)
  6. ✅ docker-compose para desarrollo local con hot-reload
  7. ✅ Seed de base de datos con 10 tareas iniciales (seed.py)
  8. ⬜ Push inicial a GitHub
✅ Verificación:
docker compose up --build → http://localhost:8000/docs
py -m pytest tests/ -v → 11 passed
🤖 Semana 2 Jenkins CI ✓ Completada

Automatizar la integración continua. Cada git push dispara el pipeline que corre tests, construye y publica la imagen en Docker Hub.

  1. Levantar Jenkins con docker-compose (puerto 8080, volumen persistente)
  2. Instalar plugins: Pipeline, Git, JUnit, Docker Pipeline
  3. Escribir el Jenkinsfile con stages Checkout → Test → Build → Push
  4. Configurar credencial dockerhub-creds en Jenkins
  5. Activar SCM polling cada 5 minutos
  6. Verificar reporte JUnit de tests en la UI de Jenkins
✅ Verificación:
Commit → Jenkins detecta cambio → 4 stages en verde → imagen en Docker Hub
⎈️ Semana 3 Kubernetes + Deploy ✓ Completada

Desplegar en Kubernetes y completar el pipeline end-to-end. Demostrar escalabilidad automática con HPA.

  1. Namespace devops-project
  2. Deployment: 2 réplicas + liveness/readiness probes (/health)
  3. Service tipo LoadBalancer (puerto 80 → 8000)
  4. ConfigMap con variables de entorno de la app
  5. HPA: escalar 2–5 pods cuando CPU supere el 60%
  6. Stage Deploy en Jenkinsfile (kubectl apply -f k8s/)
  7. Validación end-to-end + informe técnico PDF
✅ Verificación:
kubectl get pods -n devops-project
→ 2 pods en estado Running
curl http://localhost/health → {"status":"ok"}
